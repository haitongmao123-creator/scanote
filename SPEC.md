# scanote - 规格文档

## 1. 项目概述

- **项目名称**: scanote
- **项目类型**: 纯前端单页Web应用
- **核心功能**: 自动或手动识别PDF内容并命名文件
- **目标用户**: 需要处理大量扫描件的企业员工

## 2. 功能规格

### 2.1 核心功能

#### 2.1.1 两种工作模式
- **邮箱自动模式**: 接入IMAP邮箱，监听指定发件人的邮件，自动提取PDF附件并命名
- **手动拖拽模式**: 直接将PDF文件拖入软件界面进行处理

#### 2.1.2 识别方式
- **本地模式**: 使用PDF.js提取PDF文本内容，识别顶部标题（免费、防泄密、无需联网）
- **AI API模式**: 调用AI接口进行内容识别（需配置API Key和端点）

#### 2.1.3 命名规则
- **三段式默认格式**: `扫描件_提取标题_YYYYMMDD`
- **可自定义**: 三段（扫描件前缀、提取标题、日期）可任意组合和排序
- **标题限制**: 最多10个汉字
- **二段式/一段式**: 可选

#### 2.1.4 保存位置
- **默认**: 桌面
- **可自定义**: 用户指定任意文件夹

### 2.2 数据持久化
- **存储方式**: localStorage
- **保存内容**:
  - API配置（Key、端点、模型）
  - 邮箱配置（服务器、端口、用户名、密码）
  - 命名规则配置
  - 保存路径配置
- **无服务器**: 无需登录，刷新页面或换设备后需手动恢复配置

### 2.3 用户交互流程

#### 邮箱模式
1. 配置邮箱IMAP设置 → 保存
2. 开启自动监听
3. 收到邮件后自动提取PDF
4. 识别内容并按规则命名
5. 下载/保存文件

#### 手动模式
1. 将PDF拖入拖放区
2. 选择识别模式（本地/AI）
3. 预览识别结果
4. 确认或修改文件名
5. 选择保存位置
6. 确认保存

## 3. UI/UX规格

### 3.1 布局结构
- **整体**: 单页应用，垂直布局
- **顶部**: Logo + 标题

#### 字体
- **主字体**: "Inter", -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif
- **中文**: "PingFang SC", "Microsoft YaHei", sans-serif
- **标题**: 24px/600
- **正文**: 14px/400
- **辅助**: 12px/400

#### 间距系统
- **基础单位**: 8px
- **容器内边距**: 24px

- **过渡时长**: 300ms

#### 顶部导航
- Logo图标 + 应用名称
- 简洁横向排列

- 两个标签: "邮箱模式" / "拖拽模式"

#### 配置面板
- 可折叠面板
- 本地模式开关

#### 拖放区域
- 大尺寸虚线边框区域
- 拖入时边框变蓝 + 背景色变化

#### 文件列表
- 卡片式展示
- 显示: 原文件名 → 新文件名
- 预览识别结果
- 操作按钮: 下载、重命名、删除

#### 命名规则配置
- 拖拽排序
- 开关控制显示/隐藏各段
- 实时预览

## 4. 技术实现

### 4.1 前端框架
- 纯原生HTML/CSS/JavaScript
- 无框架依赖

### 4.2 关键库
- **PDF.js**: Mozilla开源库，用于本地PDF文本提取
- **拖拽API**: 原生HTML5 Drag and Drop

### 4.3 外部API调用
- **邮箱**: 使用用户配置的IMAP服务器（通过浏览器CORS代理或扩展）
- **AI API**: 支持OpenAI兼容接口

### 4.4 数据存储
- **localStorage**: 配置、API密钥、命名规则
- **IndexedDB**: 临时存储PDF文件和识别结果

### 4.5 安全考虑
- API Key仅存储在本地
- 不通过自有服务器中转
- HTTPS优先
